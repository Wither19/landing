The Rant



So I've been putting a whole lot of effort into this whole front-end web development thing. The biggest catalyst for it was a web design course at school, and through that I became quickly acquainted with the ins and outs of HTML and more importantly, CSS. However, it became apparent to me a couple of months in, that there are clear limitations to what both of these languages can do. I needed to pick up JavaScript knowledge. Unfortunately, the next web design class meant to teach JS wasn't being tought this semester. None of that mattered, because I was getting bored of CSS and many parts of my sites and projects were lacking in a certain interactivity. So despite having no initial knowledge of scripting, no tutorials or teachers to guide me, I started learning JQuery. Frankly, I can think of many reasons as to why I chose to pick up JQuery in specific.

JQuery is advertized with the tagline "Write less, do more". Based on my experience with it so far, I'd say that's no doubt the case. with vanilla JS (which I tried first), to select a specific element in the DOM you have to either:

A) thoroughly type out "document.querySelector('#id')" each and every time

B) type the aforementioned code snippet as a variable to reuse

Either way, it makes for a hassle that I wouldn't put up with. So, when I saw that in JQuery, the option IS afforded to you to make it a function for repeated use, in addition to being able to do this:

"$('#id')"

The sheer simplicity! I was so happy to see that not only was it shortened dramatically, but that JQuery harnessing previous understanding and expertise in their library is quite common. Wherever possible, the devs made the many facets of the library emulate what you've already seen and learned before in front-end, as opposed to making things complicated.

Of course, you could argue that for quite a while now, vanilla JS holds everything that JQuery does. I really don't care. I still want to use the latter. From what I've seen, in essence, JQuery does exactly for JS what Bootstrap does for CSS. Yeah, their focuses and goals as libraries are entirely different, but they both focus on saving you a lot of coding. slideToggle is a great example of it. You yourself could code a simple JS-based accordion (In MY case it would take a bit), but why would you do that, when a library's already accomplished that menial task for you? Also, I'm already in too deep in JQuery, too much effort to relearn the basics in a new language. So nah, I'm good. No JS for me. Anyway, up above is what you really came here for, my epic keyboard shortcuts lol
